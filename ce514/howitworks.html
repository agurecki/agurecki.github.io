How the Web Application Works

1. Leaflet Map and KML File Handling

The map is initialized using Leaflet, and KML files representing different rivers are loaded when a user selects a river. No river is automatically added to the map.

// Initialize the map var map = L.map('map').setView([42.3601, -71.0589], 9); // Center on Massachusetts // Add a tile layer (OpenStreetMap) L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', { attribution: '© OpenStreetMap contributors' }).addTo(map); // Load KML files var rivers = { Charles: omnivore.kml('https://raw.githubusercontent.com/agurecki/agurecki.github.io/refs/heads/main/ce514/Charles%20at%20Dover.kml'), Concord: omnivore.kml('https://raw.githubusercontent.com/agurecki/agurecki.github.io/refs/heads/main/ce514/Concord%20at%20Lowell.kml'), Merrimack: omnivore.kml('https://raw.githubusercontent.com/agurecki/agurecki.github.io/refs/heads/main/ce514/Merrimack%20at%20Lowell.kml'), Shawsheen: omnivore.kml('https://raw.githubusercontent.com/agurecki/agurecki.github.io/refs/heads/main/ce514/Shawsheen%20Andover.kml'), Squannacook: omnivore.kml('https://raw.githubusercontent.com/agurecki/agurecki.github.io/refs/heads/main/ce514/Squannacook%20River.kml'), Neponset: omnivore.kml('https://raw.githubusercontent.com/agurecki/agurecki.github.io/refs/heads/main/ce514/Neponset%20at%20Norwell.kml') }; var activeLayer = null; // Track the currently active layer

2. Selecting a River

When a river is selected, its corresponding KML file is added to the map, and previous layers are removed. Each river's reachID is defined to make it accessible when called in the function later on. Only one river appears at a time, and when one river is selected, the previous one is removed. The map zooms to the location of the selected river. If no river is selected, the map returns to the original view.

// Reach IDs for each river const reachIds = { "Squannacook": 6076039, "Concord": 6771379, "Merrimack": 6746184, "Shawsheen": 6744972, "Charles": 5867435, "Neponset": 5867537 }; // Variables to store selected river and range let selectedRiver = null; let selectedRange = null; // Function to handle river selection function toggleRiver(riverName, button) { document.querySelectorAll('.river-button').forEach(btn => btn.classList.remove('active')); if (activeLayer) { map.removeLayer(activeLayer); activeLayer = null; } if (selectedRiver === riverName) { selectedRiver = null; map.setView([42.4072, -71.3824], 8); // Reset map view } else { selectedRiver = riverName; button.classList.add('active'); activeLayer = rivers[riverName]; activeLayer.addTo(map); // Add selected river to the map map.fitBounds(activeLayer.getBounds()); } }

3. Fetching Forecast Data

The function retrieves the selected riverâ€™s reach ID and calls the NOAA API to get forecast data based on the selected river and range. If there is no data or an error occurs, an error message is returned.

// Function to fetch and display forecast data async function getForecast() { if (!selectedRiver || !selectedRange) { alert("Please select both a river and a forecast range."); return; } const reachId = reachIds[selectedRiver]; const apiUrl = `https://api.water.noaa.gov/nwps/v1/reaches/${reachId}/streamflow?series=${selectedRange}`; try { const response = await fetch(apiUrl); if (!response.ok) { throw new Error(`HTTP error: ${response.status}`); } const json_data = await response.json(); if (!json_data[selectedRange] || !json_data[selectedRange].series || json_data[selectedRange].series.data.length === 0) { throw new Error("No forecast data available."); } const streamflowData = json_data[selectedRange].series.data; const timestamps = streamflowData.map(item => item.validTime); const flowValues = streamflowData.map(item => item.flow); updateGraph(timestamps, flowValues, selectedRange); document.getElementById('forecast-container').style.display = 'block'; } catch (error) { console.error('Error fetching forecast:', error); alert("Error fetching forecast: " + error.message); document.getElementById('forecast-container').style.display = 'none'; } }

4. Displaying the Forecast Data

The retrieved streamflow forecast is displayed in a chart. The chart is updated with the flow level (cfs) and times the flow will occur.

function updateGraph(streamflowData) { const timestamps = streamflowData.map(item => item.validTime); const flowValues = streamflowData.map(item => item.flow); new Chart(document.getElementById('streamflowChart'), { type: 'line', data: { labels: timestamps, datasets: [{ data: flowValues, label: 'Streamflow Forecast' }] } }); }
